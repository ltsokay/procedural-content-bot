# Генератор процедурного контента — Telegram-бот

Telegram-бот, который генерирует игровые **карты**, **лут** (предметы) и **имена персонажей**
с учётом темы и сложности и экспортирует результат в **JSON** и **XML**.

Это рабочая реализация проекта, описанного в курсовой работе.

## Возможности

- Генерация карт на основе шума Перлина (связные области: вода, равнины, лес, горы).
- Генерация лута по таблицам редкости и тематическим словарям (common…legendary).
- Генерация имён персонажей на марковских цепях по тематическому корпусу.
- Темы: `fantasy`, `sci-fi`, `post-apocalyptic`; сложность: `easy`, `hard`.
- Управление параметрами через inline-кнопки, экспорт результата файлом (JSON/XML).

## Структура проекта (и связь с курсовой)

| Файл | Что делает | В курсовой |
|------|------------|-----------|
| `procedural_content_bot.py` | Точка входа: команды, нажатия кнопок, состояние, ответы. | класс `ProceduralBot`, раздел 3.2 |
| `session.py` | `UserSession` — параметры пользователя и последний результат. | раздел 3.2 |
| `content_service.py` | `ContentService` — выбор генератора по типу контента. | раздел 3.2 |
| `generators/map_generator.py` | `MapGenerator` — карта на шуме Перлина. | разделы 2.4 / 3.3 |
| `generators/loot_generator.py` | `LootGenerator` — лут по словарям и редкости. | разделы 2.4 / 3.3 |
| `generators/name_generator.py` | `NameGenerator` — имена на марковской цепи. | разделы 2.4 / 3.3 |
| `generators/perlin.py` | Шум Перлина (со встроенной запасной реализацией). | раздел 1.3 |
| `export_service.py` | `ExportService` — сериализация в JSON и XML. | раздел 3.4 |
| `demo.py` | Быстрая проверка генераторов без Telegram. | — |
| `tests/test_core.py` | Модульные тесты ядра (24 теста). | раздел 3.5 |

Архитектура повторяет диаграмму классов из курсовой: интерфейс (бот) → состояние
(`UserSession`) → координатор (`ContentService`) → генераторы → экспорт.

## Установка и запуск

1. Установить Python 3.10 или новее.
2. Получить токен бота у **@BotFather** в Telegram.
3. Установить зависимости:
   ```
   pip install -r requirements.txt
   ```
   > Если библиотека `noise` не ставится — это не критично: используется встроенная
   > реализация шума Перлина. Можно убрать строку `noise` из `requirements.txt`.
4. Указать токен: впишите его в переменную `TOKEN` файла `procedural_content_bot.py`
   **или** задайте переменную окружения `BOT_TOKEN`.
5. Запустить:
   ```
   python procedural_content_bot.py
   ```
6. Открыть бота в Telegram и отправить `/start`.

## Быстрая проверка без Telegram

```
python demo.py
```
Выводит пример карты, предмета, нескольких имён, а также их экспорт в JSON и XML —
удобно убедиться, что логика генерации работает, ещё до настройки бота.

## Тестирование

Модульные тесты не требуют сторонних библиотек (только стандартный `unittest`):

```
python -m unittest discover -s tests -t .
```

Тесты проверяют структурные инварианты карты, распределение редкости лута, генерацию
имён марковской моделью, маршрутизацию по типу контента и валидность JSON/XML.

### Покрытие кода

Измерить покрытие операторов можно с помощью `coverage.py`:

```
pip install coverage
coverage run -m unittest discover -s tests -t .
coverage report -m
```

Покрытие операторов модулей ядра (генераторы, координатор, экспорт, состояние)
составляет около **95 %**. Модуль `procedural_content_bot.py` (слой Telegram Bot API)
в подсчёт не входит и проверяется функционально.
